21年2月第1篇周志

发表于2021-02-08|更新于2021-02-10|周志
|字数总计:8.2k|阅读时长:24分钟

本周无甚大事，前边五天写代码跑仿真，后两天在等师兄给下一步的数据，变相给自己放了个假。

写代码时深深感觉自己太久不刷Leetcode，思维完全锈死了，统共400来行的matlab算法仿真居然写了两周，当然也跟个人懒有关，写写停停，拖拖拉拉…委实不应该。又蠢又懒，真是没救了。现在只是希望师兄能赶快把新的数据发过来，这样才能按时完成第二阶段的任务。

前几天考虑了一下个人文档阅读流的问题，设计了一套方案，旨在能够在未来几年内方便地应对各类的信息的通信控制与管理保存。

另外临近年关，只好随父母走亲访友，尽虚与委蛇之事。人情世故，沉沉浮浮，个中滋味，一言难尽，遂有遗世独立之理想。好在在附近找到了一家不错的自习室，每日晨兴，即逃到这里来，端着电脑敲敲打打，晚上回家睡觉，权当现代隐士。

此外也整理了一些数据，之前师兄提供的资料、一些部落格、一些订阅内容还有几年大学的数据，从头整理了一下。感触良深。

这周没来得及看什么书。培根的《新工具》尚可，时代意志的印记明显，归纳法是重点观测对象；伽利略的《对话》又臭又长，睡前看一会有助于安眠，也不知道是其人文笔太差还是翻译不周。17世纪到19世纪是科学史上极为重要的阶段，在这一时期的各个领域中，科学逐渐取得了对宗教神学与常识的完全胜利，有必要仔细观测这一飞跃。

周末两天一直在陪妹妹玩，老妹在某些方面表现出的老成令人畏惧…看着动若脱兔，可可爱爱，心思却比头发还细，能从别人的一举一动中觉察他人的想法。有的人擅长伪装，有的人擅长观察，大概老妹就是这种擅长识人的人吧。跟她在一起，总感觉自己被看穿了呢，尽管她只是一个5岁的瘦瘦的小女孩。

嗯？

今天想说的东西有点多呢。

也许是因为太孤单了吧，自习室这边基本没什么人能说得上话，回家后跟父母也没什么好说的，也就只好用文字记录一下这副皮囊的所见了。

写点无关的东西吧。

这周，过去的高中同学们不约而同地遭到了短信验证码轰炸，基本一天两次，中午一次晚上一次，有时持续百来条验证码。算是略略体会到了当年陪都重庆的市民每日躲轰炸时的感受。华为短信拦截不能拦截验证码，完全派不上用场，当网友质问客服为什么验证码类型的短信不能拦截时，客服回复是为了防止用户收不到验证码，给正常生活造成干扰…啧…今日份的笑点。

技术上而言，IOS为短信提供了调用接口，多数短信屏蔽软件都可以很好地工作；Android就没这么幸运了，系统层级上安卓禁止除了默认短信应用外的其他应用来读取短信，本意是保护用户隐私，但实际上造成了需要root并且搭配NekoSMS之类的框架，才能补强掉制杖的华为短信的情况。当然解决方法也很简单，更改短信软件就是了，kick the ass of 不成器的华为短信后，我换用了key message，设置好屏蔽规则，就很舒服了。

分析来看的话，应该是一些人获得了同学们的手机号，并且因为一些难以理解的心态而采取了这种行动。

分两个方面随便说说吧…

首先是技术上，没有什么门槛，通过一些简单的库中的method（比如request，基本HTTP系列）就能实现对网页内容的拉取与信息的提交，代码可以用java、python….实现。然后租一台服务器，托管到上边来跑，就可以实现24h全天轰炸…当然如果受过一些硬件环境的训练，可以考虑买一块开发板，搭载linux操作系统（树莓派或者香蕉派，举例，都略微有点贵（总得过100））？接上网线完事。或者搞一块能跑嵌入式RTOS的板子（感觉这个的方案好多…）另外配置一些外设，嗯其实这种简单的功能，操作系统也不是必须的…要是成本导向的话，干脆搞一块8051，然后搭配wifi模块，连上家里的路由器，省电省钱又省事（50以内）。

虽然啰嗦了这么多，但最好的方式还是租一台服务器，灰常的便宜（几乎免费），可以通过虚拟手机号多次注册，薅免费使用期限的羊毛。所以相当于两个部分的工作。

  1. 虚拟手机号收发管理，尽管试用期限一般都会有一两周，但是也有必要自动管理，即将到期时，服务器上托管的代码会申请虚拟身份，从而将自己迁移到其他服务器来运行，从而做到无限制轰炸。（李梅点赞）
  2. 验证码轰炸，之前介绍过。

这种行为作为技术练习尚可，用它来攻击别人就太不荣耀了。

为什么会有人有这种想法呢？

尽管完全不理解，但这是事实，还是得努力理解。

报复吗…生命进化的本能是趋利避害，那么如果外界造成了损害，正常的反应应该是逃避吧…直到逃无可逃，才会选择反戈一击。

我觉得，把同学们的电话号码收集起来输到程序里的过程也挺麻烦的，几十个呢。

是谁做的呢，无非两种可能，外人与内鬼罢了…

嗯……

想不懂，实在是想不懂，简直搞不清到底怎么回事。

这是本周的周志。

费县端电脑自闭地点指南

未完工……

寒假进入中期了，要肝进度了，另外由于被父母嫌弃，有点不想待在家里，所以在找费县比较好的端电脑自闭的地方。

首先要考虑一些咖啡厅和快餐店，但是咖啡厅好像不大支持吃午饭或者晚饭的功能，所以从快餐店入手。

选择其实蛮多的…评测结果记录如下。

要求

  1. 有插头，能给电脑充电
  2. 有wifi
  3. 有卫生间
  4. 生活费不能太超标
  5. 饭比较好次就好了
  6. 人不是太多，比较安静就好了

前三个要求是基本的，后三个为加分项。寒假生活费按照1500 per month来计算，电脑的无插座续航时间记为2-3h，充满电需要1h。

肯德基（百姓超市店）

总体评价：emmm…比较神奇的一家店，除非PN结雪崩击穿大脑短路，否则我是不会去了。

优点：附近卫生间比较好找？另外服务态度不错。

缺点：

  1. 四面墙有两面主要是玻璃，也就没有插头，有一面是前台…有插头，但是都给了炸薯条的机器，还有一面也没有肉眼可见的插头。
  2. 没看到有wifi…打开无线wlan一看都是各种各样的手机热点…
  3. 另外食物也比较贵，对于比较能吃的男生，一顿饭可能得50-60才能吃饱，这样给每天分配的生活费显然超标了。
  4. 食物中的一些内容可以用神奇来形容，这些内容包括且不限于：堂食中有鸡米花或者薯条，但却没有配套的叉子或者牙签，莫非要我们效仿吃手抓饭的印度雅利安人么；另外有时会出现打开汉堡的盒子后，发现一个汉堡像消化不良的癞蛤蟆一样瘫倒在盒子里的情形。强迫我们开始深度思考，我到底是端起盒子来吃呢，还是用叉子一层一层地吃呢…所以拜托,给汉堡额外包一层纸好趴。接着，因为学生寒假来做兼职的缘故，员工的工作熟练程度有点堪忧，有时会出现鸡肉炸糊的情况。最后，这里的咖啡难喝到掉san的地步，即使要保护水资源，也不能用涮锅水来泡咖啡呀。
  5. 人不是很多，我怀疑由于性价比太低，一般没什么人来……当然也有店面比较大的缘故，显得人口密度不大高。

华莱士（东方购物城附近）

总体评价：不贵，但没有配套的卫生间是硬伤。

优点：南边的墙上有四个双孔与三孔配套的插头，有wifi，生活费不会超标。

缺点：

  1. 只有洗手的地方，没有卫生间，主要由于店面太小了。
  2. 不是很好吃…也是没有配套的餐具，只好另外找服务员来要，社恐瑟瑟发抖。另外这里的橙汁是大坑，他们只有热到没法喝的橙汁，只好往里加冰块，就结果而言，会让人想到黄胆汁（或者黏液）。此外，店员向汉堡里加沙拉酱的技术只能用失败来形容，每每出现霸气侧漏的汉堡（咱暗暗将其喻为呕吐的棕黄色蛤蟆），如果一定要吃，最好还是选择原味汉堡（没有沙拉酱），但即便如此，吃起来的感觉也是一言难尽：面皮与肉并不松软，有一种橡胶的口感…
  3. 人比较多，小孩子也比较多。现在是周六，在笔者背靠着墙，敲打键盘的时候，咱的左翼右翼与前方均被精力过剩的小孩子与他们的家长包围了（很好的钓野伏战术嗯）。不过只要专注起来，配合一些降噪器材，基本也注意不到时间与空间的变化的吧。

最高鸡密

总体评价：有插头，有wifi，店面比较小，有奇奇怪怪的背景音乐，感觉不适合在这里待太久。

关于RSS的一点碎碎碎念

RSS（简易资源整合）是很棒的咨询获取的工具，可以有效地应对海量的垃圾信息。

对于资源获取，有这几个要求：

  1. 全平台，无论是win也好，linux也罢，还是各种移动平台，都应该能够随手可得。
  2. 网址不限，xml也好，html也罢。无论怎样，都应该在我们的订阅菜单中能占有一席之地。
  3. 高度自定义的规则，硬核玩家的上上之选。支持高度的自定义与资源的整合，既然我们具有比较好的code能力，干嘛不运用呢？

实际上就只有两个选择了

  1. Inoreader
  2. 自建TT-RSS

其实…我比较喜欢自建的思路…这样比较安全，而且成本也会低一些，对于没什么闲钱买贵的要死的PRO版本的穷学生来说再好不过了（看起来还是得买域名搞服务器啊话说，没钱永远是硬伤唉）

Markdown格式没有公开标准，也没有标准机构，因此目前稍有些混乱。主要包括两个等级的标记：

Markdown工具

虽然Markdown各家支持不是很统一，但因有比较公认的多数标记格式，同时，对于日常一般的文本编辑，这些标记功能基本够用，所以，催生了不少优秀的Markdown文体编辑工具，主要可以分为这几类：

  • 各平台的文章编辑器：公众号、知乎、头条、简书、CSDN等。
  • 本地APP：MarkText、Typora、Notable、vnote等。
  • Web APP：mdeditor等。
  • 渲染库：如javascript编写的渲染器stackedit (https://github.com/benweet/stackedit)。

Markdown编辑工具发展阶段

Markdown编辑工具主要经过3个阶段的发展：

  1. 第一阶段：纯文本编辑器编辑Markdown，然后由第三方渲染工具将其渲染成HTML。此阶段，我们无法在编写Markdown文件时看到渲染的结果，因此不利于修改格式错误。
  2. 第二阶段：Markdown编辑器内置实时渲染窗口。因此编辑器一般分为两个窗口，左边是编辑纯文本Markdown，右边窗口即时显示渲染成HTML后的结果。大大降低了初学者的门槛。
  3. 第三阶段：所见即所得（WYSIWYG）编辑器。这些编辑器将Markdown纯文本编辑与实时渲染统一成一个窗口。实现编辑与渲染查看的融合，大大提高了编辑效率。

哪个是最好用的所见即所得Markdown编辑器？

所见即所得、同时又是跨平台（Windows、Linux、macOS）的Markdown编辑器，目前最火热的主要有如下几个：

  • Typora。非开源，Beta期间免费，比较成熟。
  • MarkText。开源，免费，社区活跃。
  • Notable。开源，免费，软件界面比较适合笔记管理。

設定你的免費帳號

在 ClickUp 開一個免費帳戶真的很簡單，只要一個 e-mail 帳號就行了，也不用綁信用卡 (將來想升級成付費帳戶時才需要付款方式的資料)。

開了免費帳戶後，就會進入幾個簡單的設定步驟。

\1. **Customize your avatar 設定用戶形象：**你可以上傳一張大頭照、你的個人公司 logo (左邊)，或是使用名字的縮寫當用戶形象 – 使用名字縮寫的話，可以選擇一個喜歡的顏色來代表你 (右邊)。

\2. Choose your favourite color 選擇操作界面的主題顏色：ClickUp 的界面基本上是很乾淨的白色，這個顏色主要會出現在按鈕、強調等地方。

\3. **Do you want to add any personal apps for yourself 連結其他線上工具：**這部分可以先跳過，因為之後隨著你的使用習慣，隨時都可以很輕易地連結其他工具，不用一開始就設定好。

\4. **選擇你是個人使用，或是團隊使用：**先選擇「個人使用」就行了，上手之後，將來若有需要加團隊再改。

\5. Pick a template 選擇模板：ClickUp 為各行各業的人，都設計了不同的模板懶人包，節省從零開始設定工作平台的時間。第一次使用者，我建議選最上方的 Quick start template 就好了，簡單明瞭，之後再根據自己的工作習慣調整。

到這邊，你會看到界面長得像下圖：

其實 Quick Start 這個模板的一連串待辦事項，就是一些最基本的操作介紹。可以一個個點開看裡面的說明跟著玩，就像新買的電玩遊戲，總會有一個「試試各種能力」的環節。懂了就可以刪掉或改成你自己的事項。

一時還無法掌握也沒關係，下文我都會談到重點。

在這個畫面中，我建議大家先練習「調整顯示方式成你喜歡的樣子」：如下圖，按下右上角紅色圈起來的那個「+」，就會出現一排選單。在左半邊任何項目打勾，或取消打勾，就可以在這個條列事項畫面中，設定資訊欄位 (column)，決定有哪些訊息你想要一眼望去、不用點進去項目細節就能看到。

請先不要點開右半邊的 Enable Custom Fields to create new columns。那是開放你自製想要的 task 標籤方式，可以是文字、選單、標籤等，花樣很多。但免費版限制 100 uses (設立、更改、刪除都各算一次 use)，而且其實左半邊的選項就已經很多了，所以一開始還不熟悉時先不用急著自製。

我建議先加上 Start date、Status 這兩個欄位，後面的教學會用到。然後，拖曳各個欄位到想要的顯示順序。

欄位加好之後，可以再進一步調整排列、顯示每一筆資料的方式：

右上角的「・・・」按下去，會出現下圖的小選單。我通常會選擇把 closed task 和 closed subtasks (即已完成事項) 關掉不顯示，畫面會比較乾淨。

旁邊的 Subtasks，可以依照個人喜好，決定要不要顯示。

而 Group by 可以設定每一筆待辦事項的排列方式，根據你的目的和邏輯設定就行了。例如 Group by due date 的話，就會從最急的往下排到最不急的。

調整完畢後，記得按下畫面上方中央的紅色 Save for everyone (如果將來你有團隊，也可以選擇只把這種排列顯示偏好存給你自己，不動到其他人看到的畫面)。忘記存取的話，離開再回來這個清單，樣貌又會回到之前的樣子！

如果待辦事項很大一串，但你只想看某一類內容，也可以用左上角的小漏斗 Filter by 篩出最相關的事項，或用視窗右上角的放大鏡，關鍵字搜尋整個 ClickUp 空間。

上手關鍵：層級管理概念

ClickUp 是一個 task management 線上工具，所以要用得順手，得先了解它整理資料的邏輯：層級管理概念 Space > Folder > List > Task > Subtask。

最高層級是 Sapce (免費版可以開 5 個)，舉例來說，可以開工作、學習計畫、私人生活、部落格等四個空間。*Sapce 的**「・・・」*按下去，選擇 Space setting 可以幫它改名字、選擇小圖示和顏色等等。

每個 Space 底下可以按「+」開無限制的 Folder 檔案夾 (如上圖中的 Quick Start)，其下再按「+」開無限制的 List 清單 (如上圖中的 Quick Start Template)，而每個清單列表中，再按「+ New task」開無限制的 Task 事項。

Folder 和 List 的「・・・」按下去，可以點連結圖示，分享給別人看到。也可以點筆圖示更改名稱。而 List 左邊的虛線圓圈，點下去可以設定 List 的顏色。這對於設定行事曆很有幫助，之後再細談。

如何用 ClickUp 整理待辦事項

在 ClickUp 界面的右下角有一個很明顯的「+ Task」按鈕，按下去就能建立待辦事項，要記得指定你要將這個事項放在哪一個 List 中。

你也可以先進入某一個清單列表，按下列表畫面中的「+ New task」開新事項。打下標題就可以按「SAVE」。

然後按項目標題，進入項目細節的畫面 (如下圖)。左上角可以看見檔案路徑，右半邊灰底的部分，有你每一次的更動歷史紀錄。

你可以在 Description 速記一些備忘內容 – 建議用英文寫，因為目前這裡的中文輸入常常有 bug 會無法換行，中文化的使用者體驗優化還在進行中。

然後，按右上角 DUE DATE 進入設定事項的起訖時間，甚至利用 Set recurring 選項把事項設定成定期重複出現 (例如每週一都要跟老闆開會、每週三都要寫一篇報告、每週五下班要去買菜)。如果只設定 Due Date，之後在行事曆上就會在截止日期時間顯示項目標題。如果有 Start Date 也有 Due Date，之後在行事曆上就會顯示時間區塊。

每個事項中，還可以設定 Subtask 子事項和 Checklist，以及加入夾帶檔案 (如下圖)。

子事項的用途，就是例如在「星期五開會」這個項目底下，要記得「星期三就要做好簡報寄給老闆確認」，那就可以設定一個子項目，因為算是同一件事，但有不同的截止日期。

而 **Checklist 就更好用了！特別適合用在「有很多步驟」的事項，因為它設定一次之後，可以存成範本 (上圖中的 Save as Template)，下次再開類似事項，可以直接套用，再也不怕漏了一步！**像我的每一個客戶服務 program 都要記得簽約、確認款項、開發票等。又例如一個 Youtuber 的「發布新影片」事項裡，可能包含了寫稿、佈置場景、錄影、剪輯、上傳、分享到臉書… 等一連串很細的步驟，每次都要重新想又怕疏漏，而且其中有些步驟可能不是他本人做，但又沒必要把每一個步驟都變成獨立的待辦事項，變得東一塊西一塊，每次都要重想重設定。這時，就可以用 Checklist，並且把別人該做的步驟指定給負責人。

最下面的 Attachments 可以上傳相關檔案。免費版 ClickUp 空間限制 100mb，所以其實可以在上面 Description 的地方，貼你的 Google Drive 等雲端檔案連結就好。

如何用 ClickUp 管理專案進度流程

待辦事項的內容和起訖日期時間設定好了，接下來就是要有效追蹤進度！

我們回到待辦事項列表，最上方可以增加不同的 View – 這也是 ClickUp 最強大的地方！同樣的一串待辦事項，透過不同的 View (觀看方式)，將許多種不同需求整合到同一個界面！

ClickUp 預設的是 List view 和 Board view。前者就是你已經調整好顯示方式的待辦事項清單列表，Board view 則是我在這一段要介紹的「專案進度流程」管理利器。另外還有許多其他不同的觀看方式，各有用處。但**免費版可無限制使用的 List、Board、Calendar 三種就已經非常夠用了！**其他那一大串特異功能，免費版有些不能用，或者限制 100 uses (增加、更改、取消其內容都各算一次 use)，詳情請見 ClickUp 方案比較表。

我們進入 Board view，會看到預設的只有「待辦、完成」兩個陽春進程。按下 TO DO 的「・・・」，選擇「Edit statuses」設計你想要的流程。

來到這個畫面中，點下左邊 Custom 就會出現一整排預設模板。或者，你也可以用 Custom 自己設計，甚至點最下方藍色的 + Add Template 設計一個模板，之後套用在別的待辦事項中。

**設計流程非常簡單，可以改名稱、改顏色、增加進程，拖曳換位置。**根據 ClickUp 的邏輯，DONE STATUSES 意思是事項完成，但事件尚未結束，這個項目依然會出現在你的各種清單列表和行事曆中。只有 COMPLETE 才算事件結束，會從你的清單和行事曆中消失，除非你規定它必須顯示把它叫出來。我因此習慣把 COMPLETE 改名字成 CLOSED。

你可以把每一個 Status 設計成有前後順序的連續動作，例如：Idea、Open project、Check、Validate、Publish、Closed。

也可以設計成沒有因果順序關係，只是用來代表不同狀態，例如：To do、Scheduled (有約定日期)、To contact (得聯絡交涉)、To sign (得簽署文件)… 因為某些事情的特性，可能過程中會在這些不同狀態中反覆跳來跳去。

我還看過更驚人的，是某位家事達人，設定吸地板、拖地、擦窗戶等待辦事項，把這些 Statuses 設計成客廳、房間、廚房、浴室…！

設計好流程之後，就可以回到 Board view，就會看到你的每一個待辦事項就像一張張卡片，可以直接拖曳幫他們左右換狀態位置。藉由上方的 Sort by，可以改變卡片由上到下的排列順序，例如 Sort by due date 就會從最急到最不急。同樣地，改變顯示方式之後要記得按 Save。

你也可以回到待辦事項清單，去 Status 欄位修改個別狀態。或者進入待辦事項的細節畫面，在裡面更改 Status 狀態。

下次開新的待辦事項時，就可以同時設定好它的起始進程狀態了。

如何用 ClickUp 管理行事曆

前面幾步都做得很順手後，行事曆管理就變得很簡單了！

只要增加 Calendar view，就會看到所有的待辦事項按照日期時間顯示在行事曆上。**建議到 ClickUp 左側檔案列表的最上方 Everything，把 Calendar 打開 pin view，這樣一來，這個地方就變成名符其實的行事曆了！**Everything 就是能以直昇機視角，一次看到你的 ClickUp 空間中所有事件的地方。

然後，按右上角的「・・・」，選擇你想要行事曆上出現哪些細節、時間以 AM/PM 或 24H 制顯示。我建議一定要打開 Show recurring tasks，不然你設定了定期待辦事項，行事曆上卻沒有，容易發生以為自己有空其實沒空的問題。

另外，我個人喜歡把 Color tasks by 設定成 List – 還記得之前談到 Space > Folder > List 時，有提到 List 前方那的虛線圓圈可以設定顏色嗎？它的用處就在這裡！根據顏色分類，你的行事曆就不會花花綠綠的，看顏色就知道這個項目是從哪一類清單出來的，不然有時看事件標題其實分不出來。例如「咖啡約」，可以知道是公事還是朋友約，端看你設定的 List 顏色類別。

在「・・・」左邊，可以分享你的行事曆給助理或是另一半看，以月曆還是週曆顯示… 記得更改完顯示方式後，要按下上方中央的 Save。

最後，在「・・・」那一長串清單的最下面一項 Sync with calendar，點進去可以設定跟你的 Google agenda、Apple iCalendar、Outlook 連結，更方便整合所有行程安排。

這樣，一份清楚美觀，隨著待辦事項進程同步更新的行事曆就完成囉！

最後一個小 Bonus

ClickUp 其實也有一個簡單的筆記本功能，雖然不是它的主打項目，還是可以介紹一下。

在 ClickUp 界面的左下角，有一個像 word 文件的圖示，按下去就會進入下圖畫面。**按右上角 New Doc 即可開一本新的筆記本，並且指定這本筆記本要跟誰共同觀看。**除此之外，你也可以在特定的 List 裡面加上 Document view，就會有一本指定在那個位置的筆記本，而在這個所有筆記本的畫面中同樣能找到。

按 + New Doc 會出現下圖這個畫面，你可以在左側列表中加頁 (紅色圈起來的地方)，變成好像一本書一樣。筆記內容可以是段落、表格、check list 等。可能因為中文化還在進行中的關係，目前用英文輸入比用中文輸入順得多。

希望 ClickUp 也能成為你的好幫手！

我真的沒想到這篇文章可以寫到這麼長！但基本上每天都會用到的精華功能，應該都談到了。

有機會我再來專門寫一篇，我如何用 ClickUp 做客戶管理、經營社群媒體吧。對於「單人服務業」類型的顧問、教練、講師等同業，應該很有幫助！

ClickUp 真的有極高的客製化潛能，每個人依據自己的需求，都能玩出不同的花樣。期待你來粉絲團與我分享你的用法和創意，給我更多靈感喔！
